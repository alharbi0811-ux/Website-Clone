🎯 **Prompt احترافي متقدم – نظام "وضع الدراسة" داخل لعبة "ركّز" (نسخة تقنية مفصلة جدًا)**

أريد إنشاء نظام جديد داخل موقعي (لعبة "ركّز") يسمى **"وضع الدراسة"**، ويكون نظامًا تعليميًا تفاعليًا متكاملًا ومربوطًا مباشرة بقاعدة البيانات.

---

# 🧠 الهدف الأساسي:

تحويل المذاكرة إلى **لعبة تنافسية سريعة** بدون تعقيد، مع نظام يعتمد على:

* مواد دراسية
* فصول
* وحدات
* دروس
* أسئلة مرتبطة بشكل دقيق

---

# 🎨 الهوية البصرية (إلزامي):

* الخلفية: أبيض
* النصوص: أسود
* اللون الأساسي: بنفسجي

### الاستخدام:

* الأزرار: بنفسجي + نص أبيض
* التحديد: بنفسجي
* المؤقت: بنفسجي + أبيض
* Borders عند التفاعل: بنفسجي

### الأسلوب:

* Clean UI
* Gaming Touch خفيف
* Animations:

  * Fade
  * Scale بسيط
* Shadows خفيفة جدًا

---

# 🏠 الصفحة الرئيسية:

إضافة زر جديد:

* الاسم: **"وضع الدراسة"**
* بجانب زر "إنشاء لعبة"
* عند الضغط → الانتقال إلى صفحة إنشاء اللعبة الخاصة بالنظام العلمي

---

# ⚙️ صفحة إنشاء اللعبة:

## 🧩 المراحل:

### 1. اختيار المادة:

* عرض المواد من قاعدة البيانات (Dynamic)
* Cards قابلة للنقر
* عند الاختيار:

  * يتم حفظ `subject_id`

---

### 2. إعداد اللعبة:

* اسم اللعبة → `game_name`
* اسم الفريق الأول → `team_1`
* اسم الفريق الثاني → `team_2`

---

### 3. اختيار الفصل:

* Buttons:

  * الفصل الأول
  * الفصل الثاني
* حفظ القيمة → `term`

---

### 4. اختيار الوحدة:

* يتم جلب الوحدات من قاعدة البيانات حسب:

  * `subject_id`
  * `term`
* حفظ → `unit_id`

---

### 5. اختيار الدروس:

* يتم جلب الدروس حسب:

  * `unit_id`

---

## 🎯 نظام "التركيز":

### إذا OFF:

* يتم اختيار أسئلة عشوائية:

  * `SELECT * FROM questions WHERE unit_id = ? ORDER BY RANDOM()`

### إذا ON:

* عرض Checkboxes للدروس
* المستخدم يحدد دروس
* حفظ قائمة `lesson_ids[]`

---

# 🎮 بدء اللعبة:

عند الضغط على "ابدأ":

## يتم جلب الأسئلة:

### في حالة التركيز:

* `SELECT * FROM questions WHERE lesson_id IN (...)`

### بدون تركيز:

* `SELECT * FROM questions WHERE unit_id = ?`

---

# 🗄️ قاعدة البيانات (مهم جدًا):

## الجداول:

### subjects

* id
* name

### terms

* id
* name (الفصل الأول / الثاني)

### units

* id
* subject_id
* term_id
* name

### lessons

* id
* unit_id
* name

### questions

* id
* subject_id
* term_id
* unit_id
* lesson_id
* question_text
* question_image (nullable)
* answer_text
* answer_image (nullable)

---

# 🔗 الربط مع قاعدة البيانات:

⚠️ مهم جدًا:

* جميع العمليات في لوحة التحكم يجب أن تكون مربوطة مباشرة بقاعدة البيانات
* عند إضافة سؤال:

  * يتم إدخاله فورًا باستخدام INSERT
* عند إنشاء مادة/وحدة:

  * يتم حفظها مباشرة

### مثال:

* عند الضغط "إضافة سؤال":

  * يتم إرسال البيانات عبر API
  * يتم تنفيذ:
    INSERT INTO questions (...)

* لا يتم استخدام بيانات وهمية (Mock Data)
